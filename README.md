`rfp-briefing` is designed to be used as a skill within an RFP folder. It reviews RFP documents and creates a concise 1-page briefing in minutes.

The briefing takes into account all available RFP documents and consists of:
* RFP Overview
* Key Requirements
* SLA Requirements
* Evaluation Criteria
* Incumbent & Competitive Landscape
* Risks & Challenges
* Submission Requirements
* Required Content Contributors

## Benefits

The skill **reduces time-to-kick-off** and allows sales teams to **increase efficiency** when it comes to: Scoping, qualification and assessment, coordination of cross-functional internal teams.

## Instructions
1. Navigate to the RFP directory 
2. Execute skill. For direct execution, use /rfp-briefing command. Otherwise, invoke with trigger phrases like "review this RFP" or "create an RFP briefing".<br>

The briefing will be saved to the RFP folder as an .md file and a .doc file.
